# 08 — Checklist exécutable pour atteindre 100% MVP web

## Règle de pilotage

- Périmètre: **web admin uniquement**.
- Principe: aucune extension de scope pendant l’exécution.
- Objectif de sortie: MVP signable par **Produit + Ops + Tech**.

## Jour 1 — Valider les 4 scénarios critiques bout-en-bout

- [ ] Scénario 1 validé: impayé -> relance -> suspension -> immobilisation.
- [ ] Scénario 2 validé: paiement régularisé -> rapprochement -> déblocage -> réactivation.
- [ ] Scénario 3 validé: maintenance -> assignation -> intervention -> clôture.
- [ ] Scénario 4 validé: incident sécurité -> alerte critique -> traçabilité admin.
- [ ] Preuves attachées (captures, événements, logs d’audit) pour chaque scénario.

## Jour 2 — Passer les critères d’acceptation MVP

- [ ] Flux critiques passants sans blocage.
- [ ] Cohérence des statuts contrat / véhicule / paiement.
- [ ] Notifications envoyées avec le bon niveau de priorité.
- [ ] Actions sensibles confirmées et auditables.
- [ ] Dashboard exploitable au quotidien par l’équipe Ops.

## Jour 3 — Fermer les bugs bloquants

- [ ] Triage complet des anomalies ouvertes.
- [ ] 0 bug critique ouvert.
- [ ] 0 bug majeur non contourné.
- [ ] Tous les mineurs restants déplacés en backlog post-MVP.

## Jour 4 — Recette finale et signature

- [ ] Recette finale exécutée sur le périmètre MVP web figé.
- [ ] Validation Produit obtenue.
- [ ] Validation Ops obtenue.
- [ ] Validation Tech obtenue.
- [ ] Signature de sortie MVP enregistrée.

## Jour 5 — Gel de périmètre et bascule backlog

- [ ] Gel de périmètre confirmé (aucune extension en cours).
- [ ] Demandes hors MVP classées en backlog post-MVP.
- [ ] Priorisation du backlog post-MVP publiée.
- [ ] Plan de suivi run opérationnel partagé à l’équipe.

## Gates de sortie (DoD MVP 100%)

- [ ] 4/4 scénarios critiques validés avec preuves.
- [ ] 5/5 critères d’acceptation validés.
- [ ] 0 critique ouvert et 0 majeur non contourné.
- [ ] Signatures Produit + Ops + Tech obtenues.
- [ ] Périmètre gelé et backlog post-MVP actif.
